# Is Typing - 运行指南 (Run Guide)

为了让整个链路跑通，你需要分别启动三个部分：**服务端**、**手机网页端**和**PC 客户端**。

建议在三个独立的终端窗口中运行以下命令。

---

### 1. 启动信令服务端 (Backend)
负责转发手机和电脑之间的实时消息。
- **目录**: `/backend`
- **命令**: 
  ```bash
  cd backend
  node server.js
  ```
- **端口**: 默认 `3000`

### 2. 启动手机网页端 (Web)
模拟手机端的输入界面。
- **目录**: `/web`
- **命令**:
  ```bash
  cd web
  npm run dev -- -p 3001
  ```
- **端口**: `3001` (指定端口以避免与后端冲突)
- **预览**: 在手机浏览器中访问 `http://你的内网IP:3001` (需在同一网络) 或在电脑上打开调试。

### 3. 启动 PC 客户端 (Desktop)
负责显示状态和模拟键盘输入。
- **目录**: `/desktop`
- **命令**:
  ```bash
  cd desktop
  npm run tauri dev
  ```
- **说明**: 首次启动会拉取 Rust 依赖并编译，可能需要几分钟时间。

---

### 🧪 调试流程
1. 确保 **Backend** 先启动。
2. 启动 **Desktop**，观察终端日志中是否出现 `SocketIO Connected!`。
3. 启动 **Web**，点击 `CONNECTED` 后在输入框输入文本并点击 `SEND`。
4. 观察 **Desktop** 的 Rust 终端日志，看是否收到 `Received Text Event`。
